How Buddycloud Works

The simplest way to describe Buddycloud is that you GET and PUT and DELETE and UPDATE content. These changes are automatically pushed to and shared with users on other sites running Buddycloud.

What could this content be?

This content is ATOM formatted posts, it's chat messages, it's any kind of file, likes, anything that helps communication and if it's not there, adding new content handlers is easy. Buddycloud has a complete set of calls that are accessible through three different APIs.

APIs

All the APIs let you shove data into and pull data out of Buddycloud, synchronise clients with Buddycloud in an easy to understand way.

Buddycloud is an open protocol with different API implementations

  • Rest based API
  • Buddycloud.JS (Javascript against a REST API)
  • XMPP-FTW (pure Javascript)
  • Stanza.io plug-in.(in development)
APIs are covered in more detail on the Buddycloud Software page.

Buddycloud Channels

Where is the API sending the DATA?

The data is going to different services. The most common one is the Buddycloud Channel server. To describe that we need to explain what a channel is. A channel is much like your Facebook feed or Twitter stream. You post content about what you are doing. Or status updates, or your Geolocation client posts latitude and longitude updates, it could be where you post your public key. Basically it's a way to push data to people that are interested in it and have them comment on it.

You could also use a channel to programmatically share states between applications. For example your game could synchronise state between clients and players by posting to an application specific channel.

Buddycloud Services

The Buddycloud server is the most common service in Buddycloud. But there are other services too. For example:

  • the media server, lets you upload media and share it with the followers of a channel or the whole wide world.
  • The push notification service tells your followers, via mobile notifications and/or email about new content.
  • There's a service that helps you connect with friends from other social networks called the friend finder service.
  • There's a content recommendations engine that uses scalable machine learning to suggest similar content ("if you liked this content, perhaps you would also like ... If you like these channels, perhaps you would like these similar channels).

And a growing range of new services.

Distributed Design

How do all these services intercommunicate?

The magic about Buddycloud: it's completely distributed. For you this means that you can connect to, find, share and exchange updates with other Buddycloud users on your own Buddycloud server, or other sites also running Buddycloud.

Network Transparency

Imagine the paradigm where Gmail users can only email Gmail users, Yahoo mail users can only email Yahoo users. Hotmail only Hotmail users. You expect to be able to email users on other sites. And yet this is how existing social networks [don't] inter-operate. Buddycloud blows this notion apart by introducing complete network transparency. Users on any Buddycloud site can contact users on any other site. This is revolutionary!

Synchronising Data

All this network transparency means that it's now possible to build social services where one organisation's users can connect share and update information with other organisations'.

The Buddycloud service takes care of this magically behind the scenes: making sure that all the right content is in the right people's channels on the right domains, with the right permissions, in realtime.

Publish-Subscribe

So what is a channel really?

A channel is a collection of publish-subscribe nodes. Each channel has a set of followers, who receive updates when new posts are created. For example, when you change your status node to "I'm happy", all your followers are pushed an update if they are online.
