Buddycloud Features

Imagine what you can build with the Buddycloud ecosystem.

Chat, Channels, Video

Add file sharing and soon WebRTC calling) between users. The Buddycloud stack takes care of sharing, data synchronisation between users and image previews.

To add fileshareing, simply call the upload API and, automatically, the binary object is shared with the right people.

No heavy lifting required

Develop your app more quickly by using simple abstractions for common tasks. Buddycloud does all the heavy lifting for you: simple and easy to and make common tasks as easy as possible to integrate.

Reduce app startup time by using Buddycloud's one-shot synchronisation call. It automatically works out what new information needs to be pushed to the mobile client and your users benefit from a fast time to glass.

Mobile friendly

Now you can develop messaging that plays nicely with your users' mobile battery and data plan. When you integrate Buddycloud you benefit from a battery-friendly push notification service and pre-processing and sorting of messages on the server so that your user's batteries last longer and you can do less integration work.

Your app can configure the type of push notifications that should wake it up. Never poll for new messages again.

Distributed design

Plug into a network of existing Buddycloud-enabled sites and users. Buddycloud's distributed design makes sure the right data is synchronised between users and sites.

Your users can connect and share with other sites also running Buddycloud. For example, a team collaborates and shares documents with a supplier on a remote Buddycloud site. Buddycloud ensures the confidential information stays behind your firewall.

Scalable

Scale up with ease when your application gets popular.

Scaling the the Buddycloud backend to to meet user growth is as simple as spinning up new server instances. Also, adding Realtime CDNs like Fanout is as simple as editing a config file.

Open Source

Use the flexibility of Buddycloud's fully open source code to extend and customise freely. All Buddycloud software is released under the Apache 2.0 licence.

Open protocols

Building on open protocols lets you choose from among competing Buddycloud server and client implementations. Buddycloud works closely with the W3C, Mozilla Foundation, and XMPP Standards group to build the strong foundation for an open communication ecosystem.

Your project needs a new type of message: Buddycloud uses ActivityStrea.ms for post types and the ATOM syndication format so you can simply extend the code and use existing libraries.

Designed to be Secure

Build your app on the strong foundation of auditable client libraries, strong client to server crypto and verifiable server code. Secure by design.

Secure communication needs open, inspectable code. Benefit from a community that cares about getting security right (Buddycloud is used by security sensitive agencies). No backdoors, just strong security by design.

Support

Get the support you need. We have a dedicated support team ready to help you ramp your new Buddycloud site and keep it running reliably.

Community Edition
The Buddycloud community edition is open source, free to download and run on your site. It aims to provide the best both worlds: rapid updates and fixes with long-term stability, supportability, and maintainability.
Enterprise Edition

The Buddycloud Enterprise Edition is integrated, tested, and includes patches, updates, SLA-based support, multi-year maintenance policies.

Designed to be modular

Design your Buddycloud stack with just the modules you need to run.

A mobile app built on built on Buddycloud would need only the Buddycloud and Push notification Servers. Later on you support users' uploading avatars, and simply add the media server.

Self-install or Hosted

Buddycloud comes in two compatible flavours: fully hosted and an install-it-yourself set of packages and instructions. Both editions use exactly the same code so that you have the flexibility to migrate your data and users between platforms by just changing the API url.
